# 法律/事實正確性審查流程（建議）

> 目的：讓這個移民規定資料庫可以「可更新、可追溯、可審計」。
> 注意：本 repo 內容是資訊整理，不是法律意見；最終法律解釋以主管機關與法規原文為準。

## 1) 三層資料模型（建議一律遵守）

1. **Sources（原文）**：`<country>/sources/*.md`
   - 只放「官方頁面文字抽取」或官方文件（PDF/法規）。
   - 檔頭固定包含：Source URL、Fetched(UTC)、HTTP status。
2. **Notes（解讀）**：`<country>/*.md`
   - 只寫流程、概念、風險、文件思維。
   - 涉及硬性規則/數字/期限/禁止/義務/罰則 → 必須在同段落附近附上 `sources/...` 引用。
3. **Index（索引）**：`<country>/99-sources-index.md`
   - 自動生成，用於追溯。

## 2) 審查優先順序（法律正確性）

- **P0（最高）**：
  - 任何「必須/不得/禁止/罰則/遣返/驅逐/無效/期限/金額」等句子。
  - 任何與身分合法性直接相關的流程（入境要件、續期、轉換、橋接/過渡、取消簽證）。
- **P1**：
  - 申請條件（資格、文件、保險、財力、英文、體檢、品格等）
- **P2**：
  - 建議性流程與文件整理方法（如 checklist、timeline、資料夾結構）

## 3) 一段話的「可審查」標準

- **可以被追溯**：附近有 `sources/...` 引用，且 sources 檔內能找到對應文字。
- **可以被時間化**：如果描述會過期（政策、表格、費用、期限），段落需寫清楚「依抓取時間」或引用 sources 的更新日期。
- **可以被分類**：標籤要能反映它屬於哪種規定（入境/工作/學生/依親/長居/入籍/合規）。

## 4) 自動化輔助（不是法律判定）

- 更新：`python3 tools/update_all.py`
- 審查掃描：`python3 tools/audit_kb.py --write-report AUDIT.md`

人工審查時，建議把 `AUDIT.md` 的 WARNING/ERROR 逐項關掉（補 sources 引用、或改寫成不宣稱硬性數字）。

## 5) 更新節奏建議

- **月更**：抓 sources + 重新生成索引。
- **事件驅動**：遇到官方改版/404/重大政策更新 → 立即更新該國 sources_manifest + 重抓 + 更新主題文。
